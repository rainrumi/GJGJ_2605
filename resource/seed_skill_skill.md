dream_seed_block_digest_resolver.gd、seed_effect.gdの内容を整理します。

# 実装方針
- メインスキルとサブスキルどちらに入れても機能する共通の seed_skill.gdを作成する。
- スキルはIDでif文を作成するのではなく、スキルごとに確立されたメソッドと、各スキルごとに設定できる変数で値を決める。
- スキルの内容はarrayで管理し、単一または細かいメソッドをいくつか組み合わせることで一つのスキルが形成される仕組みにする。

# 詳細設定
- これまで上記のスクリプトで管理していたスキルごとの定数は使用しません。したがって削除すること。

## seed_effect.gdについて
- このスクリプトは**メイン,サブスキルどちらにも使える抽象クラス**として再定義する。
- 構造はカスタムリソースに変化させる
- 内容は以下のものが実装されている
  - 変数
    - int priority
      - このスキルを発動するタイミングで、同時に発生するほかのスキルと照合する発動順を決める変数
  - メソッド
    - on_**発動条件**で命名された複数の仮想メソッド
    - 文中の**モノ**は**悪夢**と**夢の種**に分別してメソッドを作成する
    - 文中の**胃袋マス**は**消化マス**と**非消化マス**に分別してメソッドを作成する
    - **発動条件**は原則、以下の順に命名される二～三単語で構成される。
      - 単語は3~10ブロック以内が望ましい
      - on_**形容詞**_**動詞**_**名詞**
      - on_**形容詞**_**名詞**
      - on_**動詞**_**名詞**
    - (例)
      - on_fire_acid_enemy
        - 敵（悪夢）が消化されているとき
      - on_finish_acid_seed
        - 夢の種の消化が完了したとき
      - on_progress_time
        - 時間が経過したとき
      - on_selected_rewerd
        - 一定時間までにクリアしたとき
    - **例外として戦闘中や常時発動する効果が存在する。**
      - このようなパターンは以下のメソッドでまとめる
        - on_battle
          - 戦闘中に効果を発揮する
        - on_selecting_rewerd
          - 非戦闘中に効果を発揮する
## seed_effect_*.gdについて
- このスクリプトは**新規**に実装する必要がある。
- seed_effectを継承した子クラス
- カスタムリソースで構成される。
- スキル分を細分化し、一つの要素に絞った内容ごとにクラスを作成する。
  - **つまり、一つのクラスではなく、スキルパターンに応じてスクリプトを増やす必要がある**
- 内容は以下のものが実装されている
  - 変数
  - そのクラス内だけで必要な変数
    - 主にスキルごとに使用される数値的要素
    - 数値系や計算式に使用する単位は変数で保存する
    - (例)
      - アネモネ
        - float limit_up = 50.0
      - ゲッカビジン
        - int hour = 3  # 22時開始で1時終わりなので、22時から1時までの感覚である3
        - float add_damage = 5.0
  - メソッド
  - そのクラス内の処理に必要なメソッド
    - このメソッドは*seed_effect.gd*に実装されている仮想関数を基盤として使用する。
      - もしメソッド内の処理が長くなったり、メソッド化することでスコアが上がる処理はオリジナルで作成しても構わない。しかし、他メソッドから外部へのアクセスは禁じられている。
- 各種スキルはこのスクリプトを複数組み合わせて実装する。
- スクリプト名のアスタリスクの部分は**そのコードがどんな処理をするか**を表現する。
  - 命名規則は**条件**_**変化方法**_**変化するもの**
  - (例)
  - ホウセンカの場合
    - seed_effect_on_battle_hit_damage.gd
      - 戦闘中、受けるダメージが変化
    - seed_effect_on_hit_damage_change_atack.gd
      - ダメージを受けたとき、ダメージを変化
- 作成したスクリプトはscene/main/game/seed/skill内で各条件ごとにフォルダを作成し、分けてください。
