dream_seed_block_digest_resolver.gd、seed_effect.gdの内容を整理します。

# 実装方針
- メインスキルとサブスキルどちらに入れても機能する共通の seed_skill.gdを作成する。
- スキルはIDでif文を作成するのではなく、スキルごとに確立されたメソッドと、各スキルごとに設定できる変数で値を決める。
- スキルの内容はarrayで管理し、単一または細かいメソッドをいくつか組み合わせることで一つのスキルが形成される仕組みにする。

# 詳細設定
- これまで上記のスクリプトで管理していたスキルごとの定数は使用しません。したがって削除すること。

## seed_effect.gdについて
- このスクリプトは**メイン,サブスキルどちらにも使える抽象クラス**として再定義する。
- 構造はカスタムリソースに変化させる
- 内容は以下のものが実装されている
  - 変数
    - int priority
      - このスキルを発動するタイミングで、同時に発生するほかのスキルと照合する発動順を決める変数
  - メソッド
    - on_**発動条件**で命名された複数の仮想メソッド
    - 文中の**モノ**は**悪夢**と**夢の種**に分別してメソッドを作成する
    - 文中の**胃袋マス**は**消化マス**と**非消化マス**に分別してメソッドを作成する
    - **発動条件**は原則、以下の順に命名される二～三単語で構成される。
      - 単語は3~10ブロック以内が望ましい
      - on_**形容詞**_**動詞**_**名詞**
      - on_**形容詞**_**名詞**
      - on_**動詞**_**名詞**
    - (例)
      - on_fire_acid_enemy
        - 敵（悪夢）が消化されているとき
      - on_finish_acid_seed
        - 夢の種の消化が完了したとき
      - on_progress_time
        - 時間が経過したとき
      - on_clear_until_clock
        - 一定時間までにクリアしたとき
    - **例外として戦闘中や常時発動する効果が存在する。**
      - このようなパターンは以下のメソッドでまとめる
        - on_battle
          - 戦闘中に効果を発揮する
        - on_not_battle
          - 非戦闘中に効果を発揮する
## seed_effect_*.gdについて
- このスクリプトは**新規**に実装する必要がある。
- seed_effectを継承した子クラス
- カスタムリソースで構成される。
- スキル分を細分化し、一つの要素に絞った内容ごとにクラスを作成する。
- 内容は以下のものが実装されている
  - 変数
  - そのクラス内だけで必要な変数
    - 主にスキルごとに使用される数値的要素
    - (例)
      - アネモネ
        - float limit_up = 50.0
      - ゲッカビジン
        - int hour = 3  # 22時開始で1時終わりなので、22時から1時までの感覚である3
        - float add_damage = 5.0
- 各種スキルはこのスクリプトを複数組み合わせて実装する。
- スクリプト名のアスタリスクの部分は**そのコードがどんな処理をするか**を表現する。
  - 命名規則は**条件**_**変化方法**_**変化するもの**
  - (例)
  - ホウセンカの場合
    - seed_effect_on_battle_hit_damage.gd
      - 戦闘中、受けるダメージが変化
    - seed_effect_on_hit_damage_change_atack.gd
      - ダメージを受けたとき、ダメージを変化


- seed_info.gdは新たにseed_effect.gdに書かれた型のカスタムリソースをそれぞれarrayで保持できるmain_skills,sub_skills変数を保持します。
- seedが持つスキルの種類ごとにseed_effect.gdを継承したseed_effect_*.gdファイルを作成し、そこで処理に使用するメソッドや基礎パラメータの変数を管理します。
  - これはカスタムリソースとしてseed_info.gdのmain_skills,sub_skillsにアタッチできます。
- 新しいseed_effect.gdにはメインスキルとサブスキルに入れられたスキルを呼ぶことができる仮想メソッドが存在します。
  - これはタイミングがあらかじめ決められており、タイミングに沿ってforを使用して呼び出す必要があります。
  - この仮想メソッドは現在スキルを呼び出している場所があれば、そこで代わりに呼び出して下さい。
  - 呼び出し場所がない場合は、game.gdのイベントが起こりうるタイミングの場所から呼び出してください。
  - 例として以下のように使用します。
	- **消化されたとき**に発動するスキルはon_finished_acid()を呼び出します。
	- **消化ダメージを受けたとき**に発動するスキルはon_acid()を呼び出します。
  - seed_effectを継承した子クラスはこのメソッドをオーバーライドし、特有の処理を作成することができます。
- seed_effect_*.gdに保持される変数は固定された変数です。
  - スキルによって変動する変数はseed_skill_data.gdという新しいgdスクリプトをscene/main/game内に作成し、それを継承したseed_skill_*_data.gdを使用します。
	- このクラスは必要なクラスのみ作成します。
  - seed_effect_*.gdは標準的にintのpriorityを持っています。この数字はスキルの発動優先度を示します。
	- スキル発動時のforではpriorityが昇順になるような順番でスキルを発動します。
	- 現状、すべてのpriorityは100にしてください。
- seed_effect.gdとその子クラスのgdはscene/main/game/seed内にskillフォルダを作成し、その中に格納してください。
