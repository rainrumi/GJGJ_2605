# 各エリアの敵データ作成ルール
## はじめに
- 生成する悪夢は各ステージごとに管理するのではなく、データ自体はエリアごとに分けてひとまとめに管理することが望まれます。
  - 「単体ごとの敵データ」を「ステージに何を出すか」で管理するのが望ましいです。
  - 各悪夢は、様々なパラメータを持ちます。すべてのパラメータはカプセル化しますが、メソッドによってほかの悪夢から変更することが可能な設計にすることが望ましいです。
  - 悪夢が持つタグはenumで管理してください。タグが存在する悪夢はenumを指定することでタグを保持できます。
    - 複数のタグを持つ悪夢も存在するため配列やリストで持つことが望ましいです。
  - 寄生型悪夢という、ほかの悪夢の消化をトリガーとして生成される悪夢が存在します。
    - この悪夢はほかの悪夢が消化されるまでマップに存在しません。
    - 基本的に、トリガーとなる悪夢のスキルによって生成されます。
    - このような悪夢はトリガーとなる悪夢側に「どの寄生型悪夢をどのような条件で生成するか」を持たせてください。
## エリアについて
- エリアは「イリユ洞窟」「大樹リラン駐屯地」「エルメナ大学」「ルノヴァ旧市街」の４種類で分けられています。
- エリアには難易度、コンセプト、そしてST***というステージデータに分けられています。ST***ひとつにつき一つのステージデータです。
## ステージについて
- ステージデータはコンセプト、Eで構成されています。
  - 実装時はEの中のエネミーデータのみ実装すればよいです。
## Eについて
- Eの中にはE***という各敵のデータが格納されています。パラメーターはそれぞれ以下の意味を持ちます。
  - ひとつめ
    - 悪夢の形を指しています。いくつか書き方が分かれています。
      - ***マス□
        - これは***マス分横に伸びている形の悪夢です。
        - 例えば2マス□なら1×2、4マス□なら1×4、5マス□なら1×5を表します。
      - ***マス***字
        - これは***マス使用する***字のような形の悪夢です。
        - 例えば4マスL字ならL字のようなマスの形の悪夢です。縦に3マス、横に2マスを4マスの形で作成し、L字のようにします。
      - ***マス正方形
        - これは***マスの正方形の形をした悪夢です。
        - 例えば4マス正方形なら2×2マスの正方形、9マス正方形なら3×3マスの正方形を表します。
      - **×**マス長方形
        - これは**×**マスの長方形の形をした悪夢です。
        - 例えば2×3マス長方形なら全6マスの長方形の形の悪夢になります。
        - 例外として、2×2マス+左上凸長方形のように書かれたものがあります。
          - これは2×2マスの正方形の左上に1マス追加した形をした悪夢です。
    - HP
      - この悪夢が持つ体力です。
    - AT
      - この悪夢が持つ攻撃力です。
    - SK
      - この悪夢が持つスキルです。内容も実装してください。
      - すべての各悪夢ごとにメソッドを用意するのではなく、似たようなスキルはメソッドを使いまわせるようにスマートな設計にすることが望まれます。
    - TG
      - この悪夢が持つタグです。enumで管理し、viewする際に文字列へ変換してください。
      - **ここに「寄生型悪夢」と書かれている悪夢は、最初は存在せず、ほかの悪夢によるトリガーで生成されます。**
        - **中には悪夢間でフラグが立っているときはループして生成されるものもあります。**
          - **この仕様を考慮したうえで、動的に生成できるような管理方法を行ってください。**
    - PR
      - この悪夢が持つパラメーターです。
      - パラメータは、基本的にこの悪夢のスキルについて確認したい要素が書かれています。
      - ここには***（**）のようにテキストが書かれています。
        - 実装する際は***（**）というように表示することが望まれます。
        - 例えばSK 隣接するモノが提供する効果量を2倍にする。、TG、PR 隣接しているモノ（増やした効果量）なら以下のようになります
          - 隣接しているモノ（○○の悪夢（隣接するモノの体力を200増やす。））
