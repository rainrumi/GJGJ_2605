# プロジェクト概要

## エンジンとプロジェクト形状

- Godot設定バージョン: `config_version=5`。
- 宣言されている機能タグ: `4.6`。
- メインシーン: `main.tscn`。
- ウィンドウビューポート: `1280 x 720`。
- ストレッチモード: `canvas_items`。
- レンダリング設定にはWindows D3D12が含まれる。
- 物理設定では3D向けにJoltが指定されているが、実際のゲームは2Dである。
- Webエクスポートプリセットがあり、`../Build/index.html` へ出力する。

## ゲームループ

  ## 勝利条件
  - 朝までに敵の悪夢をすべて消化する

  ## 敗北条件
  - 敵の悪夢をすべて消化できずに朝を迎える

  ## 主人公
  - 主人公は直接敵を「攻撃」できない。バクなので。
  - あなたは悪夢を食べるバクだが、特別なあなたは悪夢を食べるときに「消化」する必要がある。

  ## 敵
  - 敵は「悪夢」を所持しており、それを「食べ」てすべての悪夢の「消化」が完了すると倒すことが出来る。

  ## 悪夢
  - 「悪夢」には種類があり、「大人に追われる悪夢」「落下する悪夢」「仕事が終わらない悪夢」など、様々な悪夢が存在する。主人公はそれらを「食べる」と胃の中で「消化」を始める。「消化」している時は悪夢に紐づけられた様々な「代謝」が発生する。大抵のものは効果時間を終えると消化が完了する。
  - 悪夢に付属するパラメータ
    - 消化コスト
      - 消化にかかる時間。所謂体力的なもの。これがゼロになると消化完了
    - サイズ
      - 満腹度を占有する量。この数字分満腹度を占有する
    - 消化時の代謝
      - 消化時に発生する効果。悪夢によって違う
    - 攻撃力
      - 食べられていない悪夢がバクを攻撃する時のダメージ量
    - スキル
      - 一部の悪夢が所持しているパッシブ効果
      - 特殊効果。ボスなどの強敵にのみ存在

  ## バトル制限
  - １バトルの中には「時刻」が存在する
  　夜の22時から始まり、朝の6時になるとバトルが強制終了する。
  - 「時刻」は「消化」または「休憩(後述)」で(現状は)30分ずつ進む。

  ## 主人公のメインパラメータ
  - 主人公は「満腹度」「体力」をメインパラメータとして持っている。
  　満腹度→悪夢の消化を始めると占領されるコスト
  　体力→数字的な体力。消化中のダメージで減る。ゼロになると少し体力回復して時間が1時間進む「休憩」を行う

  ## スキル
  - その他に様々な効果を発揮できる「スキル」を持っている。
  　「食べる」で一度に二つ以上の悪夢を食べれる
  　消化速度を一時的に早めることが出来る（通常なら消化コスト-1の部分を-2にする）
  　など

  ## 報酬-夢の種
  - 敵を倒すと「夢の種」を獲得できる。
  - 夢の種
    - 獲得すると自身に植え付けることが出来る長期的な恩恵
    - 植えた時点で花が咲き、恩恵を得られる。
    - 特殊な花と通常の花にはそれぞれ植えられる数に制限が存在する
    - 植えられる数が上限に達した状態で新しい種を獲得すると既存の花と入れ替えすることが可能

- 以下のメモは書き方例のため読まなくて良い
```
- `HUD` が `start_game` を発行する。
- `Main.new_game()` がスコアをリセットし、プレイヤーを開始位置に戻し、`StartTimer` を開始し、HUDを更新し、Mobを掃除し、BGMを再生する。
- `StartTimer.timeout` により、`MobTimer` と `ScoreTimer` が開始される。
- `MobTimer.timeout` は `mob_scene` をインスタンス化し、`MobPath/MobSpawnLocation` 上に配置し、方向と速度をランダム化してから `Main` に追加する。
- `ScoreTimer.timeout` はスコアを増やし、HUDを更新する。
- `Player.hit` は `Main.game_over()` を呼ぶ。
- `game_over()` はタイマーを停止し、ゲームオーバーUIを表示し、BGMを止め、死亡音を再生する。
```

## 現在のシーン

- 以下のメモは書き方例のため読まなくて良い
```
- `main.tscn`: ルートは `Node`。ゲーム進行、タイマー、プレイヤーインスタンス、Mob出現パス、HUD、BGM、死亡音を持つ。
- `player.tscn`: ルートは `Area2D`。アニメーションとコリジョンを持ち、ボディ衝突時に `hit` を発行する。
- `mob.tscn`: ルートは `RigidBody2D`。`_ready()` でアニメーションをランダム化し、画面外に出たら自身を解放する。
- `hud.tscn`: ルートは `CanvasLayer`。スコアラベル、メッセージラベル、スタートボタン、メッセージタイマーを持つ。
```
