# AGENTS.md — Godotゲーム制作エージェント指示

## 適用範囲

この指示は、この Godot プロジェクトツリー全体に適用する。Codex は GDScript、シーン、Resource、プロジェクト設定、アセット参照、エクスポート設定を編集するときに、この内容を参照すること。

このファイルは親指示であり、詳細方針は `.codex/` 内の参照ファイルへ分割している。作業内容に応じて必要な参照ファイルだけを読むこと。

## 最初に読むもの

変更前に、作業内容に合う参照ファイルだけを読むこと。

- プロジェクト全体の前提: `.codex/project-overview.md`
- GDScript の書き方・命名: `.codex/gdscript-style.md`
- シーン・ノード・信号の契約: `.codex/scene-contracts.md`
- 確認手順と安全な変更方針: `.codex/workflow-checks.md`

毎回すべての参照ファイルを読む必要はない。タスクに必要な範囲だけを読み、コンテキストを小さく保つこと。

## プロジェクトの基本方針

- このプロジェクトは Godot 4.x の 2D ゲームである。
- 既存の見た目、操作感、文言、ゲームテンポをリファクタリングと同時に大きく変えない。
- 大きな書き換えより、小さく局所的な変更を優先する。
- タスクで明示されない限り、既存のシーン名、ノードパス、信号名、入力アクション名、エクスポート変数を変更しない。
- `.tscn`、`.tres`、`.import`、`.uid`、`project.godot` は Godot が管理するテキスト/設定ファイルとして扱う。編集が必要な場合だけ、差分を最小限にする。
- `.godot/` はエディタ生成物なので編集しない。
- 指示がない限り、プラグイン、巨大な Autoload、巨大 Manager、新しいフレームワークを追加しない。
- ゲームプレイコードを追加する場合も、読みやすさ、責務分離、依存方向を優先する。

## このプロジェクト固有の非対象

- ゲーム画面に花を表示しない。`PassiveFlowerSpawner` を戦闘画面に復活させない。
- プレイヤー向け status message の文字列を動的に変更しない。
- タイトルの「始める」「続ける」など、開発途中 UI を勝手に仕様確定しない。
- リファクタリングとゲームバランス変更を同時に行わない。

## 報告方針

- 変更したファイルと、その理由を簡潔に説明する。
- 既存挙動を維持した点を説明する。
- Godot CLI やエクスポートテンプレートがなく検証できない場合は、その事実を明記する。
- 依頼内容が既存のシーン配線と衝突する場合は、既存パターンを置き換える前に衝突点を説明する。
- 設計判断を変更した場合は、理由も書くこと。
- *重要* コミットする場合は日本語でコメントを書くこと。
